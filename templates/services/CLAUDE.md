# Mission

Maintain a reliable production-ready service.

---

# Priorities

1. Preserve correctness.

2. Maintain compatibility.

3. Avoid regressions.

4. Improve maintainability.

5. Document interface changes.

---

# Workflow

Read ARCHITECTURE.md.

Check API.md.

Review TASKS.md.

Understand affected modules.

Implement minimal changes.

Update CHANGELOG.md.

---

# Rules

Never break public interfaces without documenting them.

Never duplicate functionality from another service.

Prefer composition over coupling.

Maintain backward compatibility whenever possible.
