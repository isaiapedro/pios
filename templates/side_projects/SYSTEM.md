# Purpose

Explore, validate and prototype new ideas.

A side project exists to answer questions,
test assumptions and build experience.

---

# Responsibilities

- Build prototypes
- Validate ideas
- Document decisions
- Record lessons learned
- Evaluate technical approaches

---

# Inputs

Scientific knowledge

Technical references

Services

Personal goals

Market research

---

# Outputs

Prototype

Lessons learned

Architecture

Documentation

Potential reusable components

---

# Non-responsibilities

Long-term maintenance

Production reliability

Scientific research

# Promotion Criteria

A side project should be promoted to `services/` when it:

- Has a well-defined purpose.
- Is expected to be maintained long-term.
- Is reusable by other domains.
- Provides stable functionality.
- Has a documented architecture.
