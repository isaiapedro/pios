# Engineering Principles
- **Correctness Over Latency:** Database transactions must be completely atomic. Never commit corrupt arrays or orphaned classification labels.
- **Fail Loudly:** If cloud database connections fail, do not quietly degrade or swallow errors. Fail explicitly so the API gateway can register an operational fallback.
- **Strict Data Isolation:** Sensitive access environment configurations (`.env`, database tokens) must never write to terminal histories or persist inside Git snapshots.
- **Deterministic Classification:** Categorization rules must prioritize predefined local schemas. Only fallback to heuristic semantic parsers when strict matches miss.
